## 1. 유사 케이스 매칭 프로세스

검수 에이전트는 아래 3단계로 유사 케이스를 찾은 뒤 비교 검수를 수행한다.

```
[Step 1] 1차 신호 추출 — 화면 유형 카테고리 결정
         ↓
[Step 2] 2차 신호 추출 — 해당 카테고리 내 서브타입 결정 → Case ID 확정
         ↓
[Step 3] 케이스별 비교 체크리스트 실행 + 일반 규칙 검수 병행
         ↓
[출력]  일반 규칙 위반 + 레퍼런스 케이스 대비 편차 통합 리포트
```

### Step 1 — 카테고리 결정 (1차 신호)

| 1차 신호 | 카테고리 |
|---------|---------|
| BottomNavigation 또는 T로고 AppBar | **MAIN** |
| 뒤로가기(←) AppBar + 동일 규격 아이템 반복 + Sticky CTA 없음 | **LIST** |
| 투명 AppBar + Hero 이미지 (AppBar와 overlap) | **DETAIL** |
| 서브 AppBar + TextField 그룹 또는 InfoTextList 섹션 + Sticky CTA | **FORM** |
| 어두운 그라디언트 배경 + AppBar 없음 + X 닫기 버튼 | **COMPLETE** |
| 딤 배경 + 하단에서 올라온 흰색 패널 + 핸들 바 | **BS (바텀시트)** |
| 딤 배경 + 화면 중앙 흰색 카드 (radius 큰 것) | **POPUP** |

### Step 2 — 서브타입 결정 (2차 신호)

각 카테고리 내 세부 케이스 구분은 [§3 케이스 라이브러리]의 **판별 신호** 항목을 참조한다.

### Step 3 — 비교 검수 출력 형식

```
## 매칭된 케이스
케이스 ID  : CASE-XX-NN
케이스 명  : [화면명]
레퍼런스   : [이미지 파일명] / Figma Node [ID]
매칭 근거  : [매칭된 주요 신호 2-3개]

## 레퍼런스 대비 편차
  ❌ [편차 항목] — 레퍼런스: [정상값] / 검수 화면: [발견값]
  ⚠️ [확인 필요 항목]
  ✅ [일치 항목]

## 일반 규칙 위반 (DESIGN_PRINCIPLES.md 기준)
  ❌ [위반 내용] — 올바른 대안: [대안]
```

## 2. 신호(Signal) 사전

케이스 판별에 사용되는 신호 용어 정의.

| 신호 | 정의 |
|------|------|
| **T로고 AppBar** | AppBar 좌측에 T자 브랜드 로고, 우측에 스캔/장바구니/메뉴 아이콘 3개 |
| **서브 기본 AppBar** | AppBar 좌측에 ← 뒤로가기, 중앙에 타이틀 텍스트 |
| **투명 AppBar** | bg=transparent, Hero 이미지 위에 float |
| **Sticky CTA** | 화면 하단에 고정된 버튼 영역 (높이 88px) |
| **Hero 이미지** | 화면 상단에 full-width로 배치된 대형 이미지, AppBar와 overlap |
| **DateGroup** | "YYYY.MM" 형식의 날짜 그룹 헤더 (bg=#F5F5F7, full-width) |
| **동사형 CTA** | Sticky CTA 버튼의 레이블이 동사형인 것 ("다음", "구매하기" 등) |
| **핸들 바** | 바텀시트 상단의 32×4px 회색 막대 |
| **딤 배경** | rgba(0,0,0,0.5)의 반투명 오버레이 |

## 4. 케이스 매칭 불확실 처리 규칙

### 4.1 복수 케이스 후보가 있을 때

2개 이상의 케이스가 신호를 공유하는 경우, 아래 우선순위로 결정한다.

```
1순위: Sticky CTA 레이블 매칭 (FORM 화면 구분에 결정적)
2순위: 화면 타이틀 텍스트 매칭
3순위: 가장 많은 신호가 일치하는 케이스
4순위: 매칭 불확실 — 두 케이스를 병렬 비교하고 편차 리포트에 "케이스 불확실" 명시
```

### 4.2 신규 화면 (기존 케이스 없음)

기존 케이스에 매칭되지 않는 화면은:
1. 가장 유사한 케이스 1개를 `부분 참조`로 적용
2. 리포트 상단에 `"신규 화면 패턴 — 기존 케이스 없음"` 명시
3. 일반 규칙 검수(DESIGN_PRINCIPLES.md)만으로 판단

### 4.3 오버레이가 포함된 화면

바텀시트 또는 팝업이 열린 상태의 화면을 검수할 때:
1. 배경 화면 케이스 + 오버레이 케이스를 각각 매칭
2. 배경 화면 검수와 오버레이 검수를 분리하여 리포트
